# Hierarchy Check

Use this checklist to evaluate whether each design variation has a clear, intentional visual hierarchy. Run this check per variation before finalizing.

---

## Primary Hierarchy Test

- [ ] There is exactly one dominant element — the first thing the eye lands on
- [ ] The dominant element is identifiable within 1–2 seconds of viewing
- [ ] Supporting elements clearly read second, not in competition with the dominant
- [ ] Tertiary elements (body text, labels, fine print) recede without effort
- [ ] No two elements compete at the same visual weight

## Hierarchy Tools Audit

For each variation, confirm that hierarchy is established through at least two of these:

- [ ] **Size** — dominant element is visibly larger than supporting elements
- [ ] **Weight** — bold/heavy type is reserved for the most important text
- [ ] **Position** — dominant element occupies a natural entry point (top-left, top-center, or center)
- [ ] **Contrast** — dominant element has the highest contrast against its background
- [ ] **Color** — brand/accent color draws the eye to the intended focal point
- [ ] **Whitespace** — dominant element has breathing room; it is not crowded by neighbors

## Reading Order

- [ ] The composition has a clear 1-2-3 reading path
- [ ] The reading order matches the intended message priority (headline before detail, benefit before CTA)
- [ ] CTA appears at or near the end of the reading path — not buried or competing with the headline

## Hierarchy Model Match

Identify which model the variation follows and verify it holds:

| Model | Stack |
|---|---|
| Promotional | Headline (dominant) > Offer/proof (supporting) > CTA (action) |
| Editorial/brand | Image (dominant) > Brand moment or headline (supporting) > Detail (tertiary) |
| Announcement | Event name (dominant) > Context (supporting) > Action/registration (CTA) |
| Product/feature | Benefit headline (dominant) > Product visual (supporting) > Feature + CTA (tertiary) |

- [ ] The chosen model is appropriate for the content
- [ ] The variation follows the model's stack without inversion

## Common Failures

Flag and fix if any of these are present:

- Two headlines of equal size — the eye has no entry point
- CTA larger than the headline — hierarchy inverted, reads as aggressive
- Body text the same size as the subheading — levels collapse
- Image and text competing at the same visual weight — neither element wins
- Every element in brand color — color loses its ability to signal priority
- Dominant element is off-center or at the bottom without compositional intent

## Cross-Variation Hierarchy

When reviewing the full set:

- [ ] Each variation has a distinct hierarchy strategy (not all identical layouts with cosmetic swaps)
- [ ] Hierarchy strength is consistent — no variation is noticeably weaker than others
- [ ] The same content priority is preserved even when layout changes (the headline stays the headline)
