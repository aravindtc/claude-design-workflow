# Spacing and Grid Patterns

## Base unit

Use a base spacing unit and derive all spacing from multiples of it.

Recommended base: 8px

Common multiples: 8, 16, 24, 32, 40, 48, 64, 80, 96, 120

Do not use arbitrary values like 13px, 27px, or 55px. If a spacing looks right at 52px, use 48px or 56px instead.

## Frame margins

Social posts (1080px):
- Safe padding from frame edge: 48–64px minimum for text
- Generous layout: 80px margins

Web pages (1440px):
- Max content width: 1200px (120px margins each side)
- Compact: 1280px content (80px margins)

Mobile (390px):
- 20–24px horizontal margins

## Grid

12-column grid is standard for web and app layouts.
Gutter: 24px (web), 16px (mobile)

Social posts typically do not use a formal grid — use zone logic instead:
- Divide the frame into thirds or halves mentally
- Anchor text blocks to one zone
- Respect consistent left margin across all text elements

## Spacing rhythm

Pick 2–3 spacing values and use them consistently throughout a frame:
- Tight: element-to-element within a group (8–16px)
- Medium: between groups (32–48px)
- Loose: between sections (64–96px)

Never mix tight spacing in one area with loose spacing in an adjacent area without a clear reason.

## Zone anchoring for social posts

Anchor zones for 1080×1080:
- Top zone: y=48–300
- Middle zone: y=300–780
- Bottom zone: y=780–1032 (with 48px bottom margin)

Use consistent left margin (48–80px) and right margin across all text in a frame.

## Whitespace

Whitespace is structure. Large whitespace zones are:
- Premium signal
- Breathing room that emphasizes the surrounded element
- A separation between content zones

Do not fill whitespace with decoration just to fill it.
