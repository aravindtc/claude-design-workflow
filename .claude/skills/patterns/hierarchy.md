# Hierarchy Patterns

Visual hierarchy is how you control what the eye sees first, second, and third.

## The hierarchy stack

Every composition needs a clear stack:

1. **Dominant element** — the first thing the eye lands on (usually: hero image, display headline, or primary CTA)
2. **Supporting element** — the second read (context, subheading, product name)
3. **Tertiary elements** — the rest (body text, labels, fine print)

If two elements compete for dominance, hierarchy breaks.

## Tools for creating hierarchy

**Size** — larger = more important. The most reliable tool.
**Weight** — bold reads before regular.
**Position** — top-left reads first in left-to-right cultures. Top center for centered compositions.
**Contrast** — high contrast (dark on light, bright on dark) reads before low contrast.
**Color** — saturated or brand color draws the eye before neutral.
**Whitespace** — isolated elements read before crowded ones.
**Scale break** — an element significantly larger or smaller than its neighbors draws attention.

## Common hierarchy models

### Promotional post
Headline (dominant) → Offer/proof (supporting) → CTA (action)

### Editorial/brand
Image (dominant) → Brand moment or headline (supporting) → Detail/caption (tertiary)

### Announcement
Date/event name (dominant) → Context/description (supporting) → Action/registration (CTA)

### Product/feature
Benefit headline (dominant) → Product visual (supporting) → Feature detail + CTA (tertiary)

## Hierarchy failures to avoid

- **Two headlines of equal size** — the eye doesn't know where to start
- **CTA larger than the headline** — hierarchy inverted, feels like spam
- **Body text the same size as subheading** — no sense of levels
- **Image and text competing at the same visual weight** — neither wins
- **Every element in brand color** — color loses its hierarchy signal
