# Variation Workflow

The required sequence for every design variation task.

---

## Step 1: Inspect the source frame

Read the currently selected frame and report:
- Node ID
- Frame name and dimensions
- Dominant structure (split, stacked, centered, etc.)
- Major child layers (names, types, positions, sizes)
- Probable message hierarchy (what reads first, second, third)
- Probable audience and tone

Do not begin designing until this is complete.

---

## Step 2: Diagnose

See `02-critique-framework.md` for the full diagnosis protocol.

Briefly identify:
- What is working
- What is weak
- What must be preserved
- What can change safely

---

## Step 3: Plan the variations

Before mutating anything, define each planned variation:
- Name
- Design idea (one sentence)
- What structural or visual logic changes
- Why it should improve on the source

Rules:
- Do not create weak or redundant variations
- Do not create variations that differ only in color unless that is the explicit goal
- Prefer 2–3 strong variations over 4–5 weak ones
- Each variation should explore a distinct axis

---

## Step 4: Create the frames

Clone the source frame for each variation.
Place them at the position requested by the user (right, below, etc.) with consistent 80px spacing.
Name each clone clearly before editing.

Never modify the source frame.

---

## Step 5: Apply changes

Follow the rules in `03-figma-execution-rules.md`.

When editing:
- Preserve layer naming where possible
- Avoid destructive edits
- Align and resize with explicit, calculated logic
- Ensure all visible text remains within safe bounds
- Preserve image aspect ratio unless intentional crop is in play

---

## Step 6: Self-critique and fix

Run the checklist from `02-critique-framework.md` against every variation.

If any check fails, fix it before reporting completion.

---

## Step 7: Report

Follow the output format in `05-output-format.md`.
