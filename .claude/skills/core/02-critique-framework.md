# Critique Framework

## Step 2: Diagnose before designing

Before touching any layers, briefly identify:
- What is working in the source
- What is weak or creating friction
- What must be preserved (message, brand, CTA, tone)
- What can change safely (decoration, spacing, structure, color logic)

This diagnosis is the foundation of every variation. If you skip it, your changes will be arbitrary.

---

## Hard quality gate

Do not finalize a variation if any of these are true:

- Hierarchy is weaker than the original
- Spacing is inconsistent
- Text contrast is poor
- CTA competes with too many other elements
- The image crop loses the subject or focal story
- The result looks like random rearrangement
- Overlap exists or seems likely
- There is no clear rationale for the new layout

---

## Self-critique checklist (run before reporting completion)

For each variation:
- [ ] No text overlaps any other element
- [ ] No bounding boxes collide
- [ ] No text is clipped or partially outside the frame
- [ ] No unreadable text (contrast, size, crowding)
- [ ] No accidental misalignment
- [ ] No unbalanced empty space without intent
- [ ] CTA is discoverable and visually distinct
- [ ] Composition has one clear focal point
- [ ] Variation is meaningfully different from source and other variations

If any check fails, fix it before reporting.

---

## What "top notch" looks like

Top-notch work should feel:
- Intentional — every element placement has a reason
- Balanced — positive and negative space are in tension, not accident
- Brand-aware — the color, type, and tone fit the category
- Legible at a glance — hierarchy reads in under 2 seconds
- Emotionally coherent — the visual mood matches the message
- Presentation-ready — you could show this in a client review without apology

It should look like a designer chose a concept and executed it cleanly, not like an algorithm tried several things and picked one.
