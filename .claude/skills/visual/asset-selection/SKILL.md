# Visual Asset Selection Engine

This system defines and selects imagery BEFORE UI is built.

---

## Goal

Ensure that visuals:
- support storytelling
- match the style
- enhance clarity
- are not decorative filler

---

## When to activate

After:
- content story is complete
- page structure is defined

Before:
- design system generation
- Figma build

---

## Phase 1 — Section Analysis

For each section:

Identify:
- purpose of the section
- emotional tone
- communication goal

---

## Phase 2 — Image Intent

For each section define:

- what the image should communicate
- what should be avoided

Example:

Hero:
- communicate clarity + intelligence
- avoid generic stock people smiling

---

## Phase 3 — Image Type Selection

Choose:

- photography (real world, human, environment)
- illustration (conceptual, abstract)
- UI/diagram (for clarity)
- iconography

---

## Phase 4 — Search Query Generation

Generate 3–5 high-quality search queries per section.

Format:

- “minimal industrial workspace natural light”
- “community collaboration outdoor garden diverse people”
- “clean medical environment soft light human interaction”

---

## Phase 5 — Placement Strategy

Define:

- hero background / side image
- inline section visuals
- cards / thumbnails
- supporting imagery

---

## Phase 6 — Output + Pause

Present:

- section → image intent
- image type
- search queries

Then:

→ STOP  
→ WAIT for user approval or edits  

---

## Rules

- images must support meaning
- avoid generic stock clichés
- match approved style
- maintain consistency across sections
