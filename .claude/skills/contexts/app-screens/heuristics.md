# App Screen Heuristics

_Stub — populate when working on app screen variations._

## Key principles

- Primary action should be reachable with thumb in the safe zone (bottom ~40% of screen)
- Navigation elements (tab bar, back button) should be consistent across screens
- Empty states need a message and an action — never just blank space
- Loading states should indicate progress, not just spin
- Error states need clear messaging and a recovery path

## Hierarchy model

- Screen title or context indicator (top)
- Primary content zone (middle)
- Primary action (bottom, prominent)
- Secondary actions (lower visual weight, accessible but not competing)
