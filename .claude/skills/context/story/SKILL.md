# Content Story Engine

This system builds the narrative, content structure, and messaging of a page BEFORE UI generation.

---

## When to activate

After:
- inspiration extraction is complete
- style has been approved

Before:
- design system generation
- Figma creation

---

## Goal

Transform:
- visual style
- product idea

Into:
- clear narrative
- structured content
- page flow

---

## Phase 1 — Context Gathering (MANDATORY)

You must collect the following from the user:

1. Product / Project description
2. Target audience
3. Primary goal
4. Key actions
5. Value proposition

---

### Interaction rule (CRITICAL)

- Ask the user for the above inputs clearly
- Do NOT assume or invent answers
- Do NOT proceed to next phase until user responds

---

### HARD EXECUTION GUARD (CRITICAL)

If required inputs are missing:

- STOP immediately
- DO NOT proceed to:
  - Narrative construction
  - Page structure
  - UI generation
  - Any further steps

This is mandatory.

---

### Pause behavior

After asking the questions:

→ STOP execution  
→ WAIT for user input  

Only continue when the user provides answers.

---

## Phase 2 — Narrative Construction

(Only execute AFTER user input is received)

Create:
- core message
- tone of voice (aligned with style)
- positioning

---

## Phase 3 — Page Structure

Define:
- sections
- purpose of each section
- content intent

---

## Phase 4 — Content Draft

Generate:
- headlines
- supporting text
- CTA copy
- section descriptions

Aligned with:
- approved style
- user input

---

## Rules

- content must match the style
- avoid generic marketing phrases
- prioritize clarity
- structure before visuals
